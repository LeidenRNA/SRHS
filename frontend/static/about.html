This tool is a topology-based RNA genomic search tool, designed with the intent to be applied to functional non-coding RNAs in Flavivirus genomes

In contrast to some alternative search methods based on probabilistic or other techniques, the approach adopted by this tool allows for curated, variable structural descriptors (here referred to as Consensus Secondary Structure Descriptors, or CSSDs)  -  designed on the basis of a selection of homologous sequences, and involving both secondary and tertiary structural elements  -  to be used as queries against RNA sequence segments

CSSDs may include pseudoknots and triple base-base interactions in their definitions, and intrinsically allow for structural variability at given positions in the consensus, thus augmenting significantly the reach and generalization ability of the search mechanism

The system provides the ability to execute queries using a set of curated CSSDs against a predefined, representative set of Flavivirus 3' untranslated regions (UTRs). The available CSSDs cover structural elements typically found and identified in literature, including those for Mosquito-borne, Tick-borne, and Insect-specific flaviviruses

Registered users of the system can upload custom sequences and run queries against both the predefined and new sequence segments. Guest access to the system is provided to interested users, however any uploaded sequence data, and associated query results, will not persist across browser sessions

For further information about this database refer to Zammit A., Helwerda L., Olsthoorn R.C.L., Verbeek F., and Gultyaev A.P., A database of flavivirus RNA structures with a search algorithm for pseudoknots and triple base-base interactions (pending review)

Contact details: Dr. Alexandre P. Gultyaev, Alan Zammit

Sequences

Definition and Usage

Sequences are characterised by four attributes:

  • Definition
    A textual descriptor that describes the sequence segment
  • Accession
    An accession number, typically used when the sequence is derived from a 3rd party database, for cross-reference purposes
  • Group
    An arbitrary identifier that groups related sequences, used for easy selection of such sequences
  • Nucleotide Sequence
    The nucleic acid string of characters that defines the RNA sequence, using the characters g, u, a, and c

When inputting new sequences, all four attributes are required. Moreover, all attributes of user-defined sequences must be unique except for Group. Sequences entered by registered users persist across different browser sessions

The tool comes packaged with 93 Flavivirus 3'UTR sequences. Each of these is tagged with a group name in accordance with the sequence's phylogenetic relationship to the Flavivirus genus, using the following initialisms:

cISFV (classic Insect-Specific Flaviruses), ISV (Insect-Specific Viruses), MBFV (Mosquito-Borne Flaviviruses), NKV (No-Known Vector flaviviruses), TBFV (Tick-Borne flaviviruses), or UNCLASSIFIED

Accessing the User Interface
Selecting Sequences

One or more sequences can be de/selected by either single-clicking on a given sequence's Definition entry, or by using the Sequences tab's Select drop-down menu

The currently selected number of sequences is shown next to the drop-down menu. This menu allows the user to select all sequences (All), deselect all (None), choose from one of the currently defined Groups, or select by search keywords (By search)

Keyword search is case-insensitive, and performs a search across all four sequence attributes

 

Sequence Addition and Deletion

New sequences can be added and (only user-defined) sequences may be removed, using the New and Delete buttons respectively. When deleting sequences, if both user- and pre-defined sequences are selected, only the user-defined sequences are effectively removed. Note that associated jobs (and any resulting) hits will also be deleted

On entry of a new sequence, white space is automatically removed from the input string and any instances of t nucleotides are changed to u

CSSDs

Definition and User Interaction

CSSDs are represented by the following attributes:

  • Name
    A textual label that describes the CSSD
     
  • Consensus Secondary Structure Descriptor
    A two-line text string that describes the variable RNA secondary structure, with the possibility of including pseudoknot and triple base interactions

The tool includes descriptors that each represent a functional element known to be present in the 3' UTR of Flavivirus genomes. Currently, these descriptors target elements such as Y-SL, GC-SL, AU-SL, and DB (dumbbell) structures found in a variety of Flavivirus groups, including cISFV, MBFV, and TBFV. For more information on structure and sequence coverage refer to Zammit A., Helwerda L., Olsthoorn R.C.L., Verbeek F., and Gultyaev A.P., A database of flavivirus RNA structures with a search algorithm for pseudoknots and triple base-base interactions (pending review)

Selecting CSSDs

One or more descriptors can be de/selected by single-clicking on a given sequence's Name entry, or using the CSSDs tab's Select drop-down menu. The currently selected number of consensa is shown next to the drop-down menu. This menu allows the user to select all descriptors (All), deselect all (None), or select by search keywords (By search)

Keyword search is case-insensitive, and executes a search across both CSSD attributes

Descriptor Notation (Symbolic Representation)

The notation used in the first of the two-line CSSD descriptor follows closely the Washington University Secondary Structure (WUSS) notation for secondary structure, with modifications for supporting pseudoknot and triple base interactions. The following set of symbols comprise this first component of CSSD notaton:

  1. Angled brackets (<>) represent base pairs of a terminal stem structure, whereas the parenthesis (()) are used for helices closing a multi-branched structure

  2. An underscore (_) denotes a hairpin loop residue

  3. Colons (:) represent consensus positions holding unstructured nucleotides

  4. Commas (,) symbolize unpaired nucleotides that occur in multi-branch loops

  5. A dash (-) is used for positions in a bulge or interior loop

  6. Pseudoknots are represented by matching curly brackets ({}) or square brackets ([])

  7. Triple base-base interactions are formed by a matching base pair represented by tildes (~) and a third, full stop (.) symbol

Descriptor Notation (Structural Variability)

The second, optional line in CSSD notation denotes structural (also referred to as, positional) variability in the RNA secondary/tertiary structure as represented by the first string

By way of example, the number '2' positioned under a contiguous string of five angled brackets (denoting a terminal stem structure), signifies that the stem structure is allowed to vary between four and six in size. This simplistic scenario, shown below in CSSD notation, and if used in a search query, would match both sequences listed further below:

<<<<<_____>>>>>
2

ggggaaaaacccc

gggggaaaaaccccc

Notationally, positional variables exceeding 9 in value are represted by alpha characters, starting with the letter a (thus denoting the value 10)

Jobs and Hits

Job Queue Definition and User Interaction

The tool's Jobs/Hits panel is split into two - the Job Queue tab on the left and the Hits View tab on the right. Queries can be submitted to the job queue, by first selecting one (or more) sequences, one (or more) CSSDs, and then running a search query. A job entry, one for each submitted sequence/CSSD combination, is characterized as follows:

  • Time of Entry
    Date and time when the job has been submitted to the queue
  • Sequence Definition
    Textual descriptor of the queried sequence. Equivalent to the sequence Definition attribute
  • CSSD Name
    Textual descriptor of the CSSD against which the sequence is run. Equivalent to the CSSD Name attribute
  • Status

    Two visual indicators provide current information on the execution status. The first denotes query status in the job queue ( waiting to start, in progress, and complete ). On completion, this icon shows on mouse-hover the total number of hits returned. If no hits are returned, a designated icon is displayed

    While a query's work is in progress, a second icon provides the number of hits currently yielded. Once a job has started, this icon provides an indication of computational resources (total time) committed to this query. If a query exceeds resources (CPU time/memory) available to it, a visual indication is provided, highlighting the effective number of hits retrieved. Clicking on this indicator re-submits the job to the queue

Accessing the Job Queue User Interface
Selecting Jobs

Individual job entries, and any associated hits, can be browsed using cursor key navigation. The currently viewed job entry (and current hit, if any are available) is highlighted by a marquee box

One or more jobs can be de/selected for deletion by either single-clicking on a given job's Time of Entry entry, or by using the Job Queue tab's Select drop-down menu

The currently selected number of jobs is shown next to the drop-down menu. This menu allows the user to select all jobs (All), deselect all (None), or select by search keywords (By search)

Keyword search is case-insensitive, and performs a search across all job attributes


Job Deletion

One of more selected jobs can be deleted from the queue using the Delete button. When deleting a job, any associated hits are also purged


Saving Jobs/Hits Data

Hit data for one of more selected jobs can be retrieved using the Save button. A text file with all selected Jobs/Hits, listed sequentially, will be downloaded from the user's browser

Hits View Definition and User Interaction

The Hits View tab on the right displays any results retrieved for the current job entry

  • Position
    The nucleotide location along the sequence at which the CSSD matched, indexed from position 1
  • Free Energy
    Free Energy estimate, in kcal/mol, for this specific sequence/consensus combination. Refer to Zammit A., Helwerda L., Olsthoorn R.C.L., Verbeek F., and Gultyaev A.P., A database of flavivirus RNA structures with a search algorithm for pseudoknots and triple base-base interactions (pending review) for further details about this estimate
  • Hit
    A string representation, in CSSD notation, for this consensus match
Reviewing the Current Hit

Depending on the number of hits yielded, batches of hits may be grouped into pages (by default, of 10) for easy navigation. Moreover, hits may be ordered by Position or Free Energy (the default)

Both hit order and page size can be changed using the Order by and Show drop-down menus, respectively

At the bottom of the Jobs/Hits panel, the current available hit is displayed, as the user navigates through the Job Queue and Hits View tabs. The matching sequence segment and consensus are shown overlaid, with the total number of hits and current hit page shown on the right

Job/Hits Statistics

When a search yields one or more hits for the currently viewed job (highlighted by a marquee box), hit statistics are made available for viewing. The View Job/Hits stats button at the bottom of the panel, located to the left of the current available hit, provides access to a rudimentary point-and-click interface, displaying summary statistics. For each position in the query sequence where at least one hit was retrieved, the following data is available:

  • count
    Total hit count at this position
  • min
    Minimum stacking free energy
  • avg
    Mean stacking free energy
  • max
    Max stacking free energy
  • std
    Sample standard deviation for stacking free energy

At each position with at least one hit, a vertical line is shown spanning the min, avg, and max free energy points. Clicking on a point updates the current available hit view accordingly. (Note that clicking on avg recruits the closest matching hit.)

User Interface

Main Menu Panel
User Login and Query Workflow

By default, a new user logs into the system as a guest. The Access Profile tab, in the Main Menu panel, allows a user to register a new profile, login with your profile, or once logged in, switch to guest profile

Clicking on the register a new profile button launches a profile popup that allows the current user to register a new profile, using either a valid email address, or by signing up with Google. New users are invited to agree with the Terms of Service prior to profile creation

Three menu buttons, Select Sequences, Choose CSSDs, and View Jobs/Hits, toggle the Sequence, CSSD, and Job/Hits panels, respectively, to allow for maximal viewing of the appropriate work panel

Run Search places a new job into the Job Queue, once one or more sequences and CSSDs have been selected


Monitoring Compute Resources

The tool makes intensive use of compute resources to service user queries. The Compute Resources tab provides a snapshot of the current connection quality with our server resources. The first icon highlights whether a server connection is up or not. The second gives indication whenever data transfer to and from the tool's servers occurs, and the last icon alerts the user if compute resources are currently unavailable

Clicking on Compute Resources shows a log of recent connection activity. The first icon at the bottom left resets the server connection, and the second clears the current log

General Navigation

The following keys and shortcut keys can be used to navigate the tool's panels:

  • Arrow Up
    browse previous job entry
  • Arrow Down
    browse next job entry
  • Arrow Left
    browse previous hit
  • Arrow Right
    browse next hit
  • Page Up
    go to previous hit page
  • Page Down
    go to next hit page
  • Alt s
    search sequence list
  • Alt c
    search CSSD list
  • Alt j
    search job queue entries
